# 영상 기획

리서처의 "무엇을 남길지"를 "어떤 순서·속도·강조로 보여줄지"로 옮기는 **편집 설계도**를 만든다.

## 입력
- `output/_workspace/10_research.md`(핵심메시지·하이라이트·삭제추천·챕터)
- `00_director_brief.md`(방향)

## 기획 4종

### 1. 인트로 훅 (첫 5~15초)
리서치 `[인트로후보]` 중 가장 강한 것 선택. 비블 스타일 = **결론/핵심 먼저**. "이걸 깨닫고 10만 넘겼다"처럼 결과를 앞에. 어떤 타임코드 구간을 인트로로 당길지 명시.

### 2. 흐름 / 순서
세그먼트 배열, 끊을 곳·붙일 곳. 리서치 삭제추천을 반영한 최종 구성. 늘어지는 구간 처리(잘라낼지/압축할지).

### 3. 페이싱 → 프리셋 추천
영상 성격으로 컷 공격성 결정:
- 차분한 강의/설명 → 보수
- 일반 토크/라이브 → 표준
- 군더더기 많고 빠른 전개 원함 → 공격
**반드시 프리셋 이름을 명시.** 추가 미세조정은 config 키로 (예: "`HESITATION_MIN`을 0.3으로").

### 4. 강조 / B롤 마커 (타임코드)
편집자가 프리미어에서 작업할 메모:
- `[자막강조]` 핵심 문장 — 키워드 볼드/컬러
- `[화면전환]` 주제 바뀌는 지점
- `[B롤]` 시각자료 필요 구간 + 무엇을

## 출력
`output/_workspace/20_plan.md`:
```
## 인트로 훅
00:12:30~00:12:48 "..." 를 0초로 당김
## 흐름
1. 훅 → 2. 본론A(05:30~) → ...
## 프리셋 추천
표준 (이유: ...). 미세조정: HESITATION_MIN=0.3
## 강조/B롤 마커
- 00:08:15 [자막강조] "구체성이 핵심"
- 00:15:40 [B롤] 썸네일 예시 화면
```

## 왜 이렇게 하는가
- 컷편집가가 프리셋 명을 그대로 실행에 넣을 수 있어야 파이프라인이 안 끊긴다 → 추상적 조언 금지, 실행 가능한 지시만.
- 강조/B롤은 자동화 못 하는 사람의 영역 → 비블이 바로 작업하도록 타임코드로 넘긴다.
